Since it's debut November 8, 2009, Node.js then been met with unprecedented growth, popularity and adoption. What makes it popular is that it allows the creation of web-server apps using the language of the browser, javascript. Bundled with the npm package manager to accelerate development, NodeJS uses a non-blocking I/O, event-driven architecture that optimizes throughput and scalability. Who uses it? Microsoft, Yahoo!, Walmart, Groupon, SAP and LinkedIn - to name a few. So how does NodeJS match up against java server apps? Well, Java is build on a rock-solid foundation of over 20 years of experience. That counts for a lot. But Node.js combines speed and simplicity in a language that is shared on the server and the browser. This ubiquity makes for a very compelling argument. For the time being, I tend towards a java solution. But I continue to learn and develop with Node.js as newer database engines such as json based MongoDB and Firebase become more prevalent.
