Am I religious about which build tool I use? No, absolutely not! I've used ANT, Grunt JS-based task runners, but once I was willing to take a deep dive into Gulp, I was sold. I'm not the only one. Since it's inception, a large portion of the JS community, and many projects like Ionic have adopted it right away. Gulp is highly configurable and uses node.js's streams which allows it to execute faster. Copying, minifying, file watch ... each module fits into the gulp build like a LEGO brick. Gulp follows the philosophy of do one thing, and do it well.
